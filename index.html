Transaction: Inputs --> Ouputs

Transactions take in one or more inputs and apportion them to one or more outputs. Digital signatures are provided to verify ownership over input funds. Outputs also include scripts which allow for more advanced "Smart Contracts".

Transaction Size (Bytes)

Transactions vary in size depending on how many inputs and outputs they have and how complex the output scripts are.

Bitcoin Blocks

Blocks are mostly filled with transactions and may be up to 1 MB in size (~2000-4000 transactions).

Unconfirmed vs. Confirmed transactions: Part 1

Unconfirmed TX: Valid transaction awaiting to be added to the blockchain.

Unconfirmed vs. Confirmed transactions: Part 2

Confirmed TX: On the blockchain. Typically you also specify how many confirmations it has (how many blocks deep it is in the chain.)

Bitcoin Mining: Part 1

Miners each independently create a block selecting users unconfirmed transactions and adding other metadata.

Bitcoin Mining: Part 2

The created blocks are not "mined" until the miner correctly guesses the answer to a unique puzzle. The difficulty of the puzzle is such that only 1 block is expected to mined by the entire network every 10 minutes.

Bitcoin Mining: Part 3

After a block is successfully mined, the miner shares it with the entire network and it added to the persisted blockchain. The miner receives all the miners fees along with a miners reward which as of today is 12.5 BTC. In April 2020 this reward will drop to 6.25 BTC.

Transaction Fees: Part 1

Transactions offer a miners fee which is set by the user creating the bitcoin transaction. The higher the fee provided the more likely miners are to include the transaction in the next block. Fee is reported in satoshis (sats) and there are 100,000,000 sats / BTC

Transaction Fees: Part 2

Since blocks are limited in size, the miners care more about the fee rate as it represents how much the transaction is paying for the space its taking up (miner fee / transaction data size).

Mempool Visualized

The collection of valid unconfirmed transactions in the Bitcoin network is called the mempool. In the visual above you'll see the LIVE state of the current mempool as it grows and shrinks.

Transactions

Transactions are represented by squares and their size reflects the size of the transaction's data in bytes.

Transaction fee rate

A transaction's color is defined by its fee rate. The higher the fee rate the more a miner gets paid per amount of space taken up by the transaction in a block.

New transactions

New transactions are constantly being broadcasted across the network and are added to each node's mempool. When they are added in the visual, you'll see them drop in from the sky with a blue border.

Mined Block: Part 1

When a new block is mined it is broadcasted to the network of peer to peer full nodes. Those nodes save this block and then must remove the newly confirmed transactions from the mempool as they are no longer "unconfirmed".

Mined Block: Part 2

In the visual you'll see this as the transactions selected for the next block glow bright green and then disappear after a few seconds. Notice trends among what was selected and how the colors and sizes change after blocks are mined.
